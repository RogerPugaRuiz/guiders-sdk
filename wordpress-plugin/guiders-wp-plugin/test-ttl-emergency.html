🚨 Test de Emergencia: Eventos Legacy Sin __queuedAt

📋 Escenario de Prueba

Este test simula el problema real en producción donde hay eventos antiguos sin el campo __queuedAt que no se están limpiando y causan errores 413 Payload Too Large.

Pasos del Test:

  1. Limpiar localStorage completamente
  2. Inyectar 5000 eventos LEGACY sin __queuedAt directamente en localStorage
  3. Recargar la página para que el SDK cargue estos eventos
  4. Verificar que el SDK aplica TTL/migración correctamente
  5. Verificar que la cola se reduce a maxSize (1000 eventos)
