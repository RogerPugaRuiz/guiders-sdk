🧪 Test: Limpieza de Eventos V1 (pageUrl/pagePath)

📋 Problema Detectado

Los eventos generados por EventAggregator contenían campos V1 deprecados:

❌ Formato V1 (rechazado):
{
  "eventType": "VISITOR_CHAT_ACTIVE",
  "pageUrl": "http://localhost:8090/", ❌
  "pagePath": "/", ❌
  "metadata": { "page": {...} }
}

Estos eventos eran rechazados silenciosamente pero nunca eliminados de la cola → loop infinito.

✅ Soluciones Implementadas

  1. EventAggregator limpia campos V1: Ya no copia pageUrl ni pagePath
  2. TrackingV2Service retorna success=true: Cuando descarta eventos inválidos, permite eliminarlos de la cola
  3. TTL migra eventos legacy: Eventos sin __queuedAt reciben timestamp actual
